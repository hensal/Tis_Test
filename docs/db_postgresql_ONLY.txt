PostgreSQL setup for the Facilities API
=======================================

Purpose
-------
Use this file to create a fresh PostgreSQL database and dummy data so the current Spring Boot facilities API can be tested in Postman.

Current Spring Boot database setting:

spring.datasource.url=jdbc:postgresql://localhost:5432/vr_view_db
spring.datasource.username=postgres
spring.datasource.password=YOUR_POSTGRES_PASSWORD

Important security note
-----------------------
The facilities API permission is checked by Keycloak token role.
The current code requires this Keycloak client role:

SYS_ADMIN

So, in Postman, use an access token from a user who has SYS_ADMIN assigned in Keycloak.

The PostgreSQL role tables below are only for dummy API response data. They are not used to authenticate the user. Authentication/authorization comes from Keycloak.


PART 1: Create database
-----------------------
Run this while connected to the default postgres database in pgAdmin Query Tool:

CREATE DATABASE vr_view_db;

After that, right click vr_view_db and open Query Tool for vr_view_db.
Run PART 2 and PART 3 inside vr_view_db.


PART 2: Create tables
---------------------
Run this inside vr_view_db.

CREATE TABLE IF NOT EXISTS facilities_tree (
    facility_id BIGSERIAL PRIMARY KEY,
    parent_id BIGINT REFERENCES facilities_tree(facility_id),
    node_name VARCHAR(128) NOT NULL,
    sort_order INTEGER NOT NULL DEFAULT 10,
    tree_level INTEGER,
    is_leaf BOOLEAN NOT NULL DEFAULT TRUE,
    public_flag CHAR(2) NOT NULL DEFAULT '01',
    outer_flag BOOLEAN NOT NULL DEFAULT FALSE,
    node_description VARCHAR(1000),
    authority_setting_flg BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS files (
    file_id BIGSERIAL PRIMARY KEY,
    file_name VARCHAR(128) NOT NULL,
    file_path VARCHAR(256),
    file_type VARCHAR(64) NOT NULL,
    file_size BIGINT NOT NULL,
    storage_type VARCHAR(32) NOT NULL,
    external_file_id VARCHAR(128),
    external_url VARCHAR(256),
    external_bucket VARCHAR(128),
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS facility_images (
    image_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    file_id BIGINT NOT NULL REFERENCES files(file_id),
    image_name VARCHAR(256) NOT NULL,
    shooting_height_m DECIMAL(6,3) NOT NULL,
    ceiling_height_m DECIMAL(6,3) NOT NULL,
    guid VARCHAR(64) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS maps (
    map_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    file_id BIGINT NOT NULL REFERENCES files(file_id),
    map_name VARCHAR(256) NOT NULL,
    image_width INTEGER NOT NULL,
    image_height INTEGER NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS map_points (
    map_point_id BIGSERIAL PRIMARY KEY,
    map_id BIGINT NOT NULL REFERENCES maps(map_id),
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    x DOUBLE PRECISION NOT NULL,
    y DOUBLE PRECISION NOT NULL,
    target_type VARCHAR(128) NOT NULL,
    target_id BIGINT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS permission_master (
    permission_id BIGSERIAL PRIMARY KEY,
    permission_name VARCHAR(128) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS facility_role_permission (
    facility_role_permission_id BIGSERIAL PRIMARY KEY,
    permission_id BIGINT NOT NULL REFERENCES permission_master(permission_id),
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    keycloak_role_id VARCHAR(64) NOT NULL,
    keycloak_role_name VARCHAR(128) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS equipment_types (
    equipment_type_id BIGSERIAL PRIMARY KEY,
    equipment_type_name VARCHAR(128) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS equipment_masters (
    equipment_master_id BIGSERIAL PRIMARY KEY,
    equipment_type_id BIGINT NOT NULL REFERENCES equipment_types(equipment_type_id),
    equipment_name VARCHAR(256) NOT NULL,
    equipment_info_json TEXT NOT NULL,
    equipment_info_html TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS equipments (
    content_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    equipment_master_id BIGINT NOT NULL REFERENCES equipment_masters(equipment_master_id),
    yaw DOUBLE PRECISION,
    pitch DOUBLE PRECISION,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS annotations (
    annotation_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    yaw DOUBLE PRECISION,
    pitch DOUBLE PRECISION,
    annotation_type CHAR(2) NOT NULL DEFAULT '01',
    annotation_title VARCHAR(256) NOT NULL,
    annotation_content TEXT,
    display_expire_type VARCHAR(16),
    display_expire_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by VARCHAR(128) NOT NULL,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS hotspots (
    hotspot_id BIGSERIAL PRIMARY KEY,
    image_id BIGINT NOT NULL REFERENCES facility_images(image_id),
    yaw DOUBLE PRECISION NOT NULL,
    pitch DOUBLE PRECISION NOT NULL,
    target_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

PART 5: Spring Boot configuration
---------------------------------
In application.properties, use:

spring.datasource.url=jdbc:postgresql://localhost:5432/vr_view_db
spring.datasource.username=postgres
spring.datasource.password=YOUR_POSTGRES_PASSWORD

If you use a different PostgreSQL username, replace postgres with that username.


PART 6: Postman tests
---------------------
First get a Keycloak access token for a user who has SYS_ADMIN.
Then send these requests with this header:

Authorization: Bearer YOUR_ACCESS_TOKEN

1. Facilities list:
GET http://localhost:8081/facilities

2. Facilities keyword search:
GET http://localhost:8081/facilities?keyword=管理

3. Facilities parent search:
GET http://localhost:8081/facilities?parentId=1

4. Facilities detail:
GET http://localhost:8081/facilities/1

Expected detail result should include:
- facility_id = 1
- facility_name = 管理棟
- images list
- maps list
- map_points list
- roles list
- equipments list
- annotations list
- has_vr_image = true
- has_equipment = true
- has_emergency_annotation = true
- has_normal_annotation = true


Common errors
-------------
1. invalid_token
Meaning: Missing token, expired token, wrong token, or token is not an access token.

2. permission_denied
Meaning: Token is valid, but the user does not have SYS_ADMIN in Keycloak.

3. relation does not exist
Meaning: Table was not created in the selected database. Make sure you are connected to vr_view_db in pgAdmin.

4. duplicate key value violates unique constraint
Meaning: You inserted the dummy data more than once. Use a fresh database, or delete existing rows first.
