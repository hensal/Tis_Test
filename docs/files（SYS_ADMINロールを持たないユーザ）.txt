files（SYS_ADMINロールを持たないユーザ）
======================================

Purpose
-------
This guide explains how to prepare PostgreSQL test data and test these Excel cases:

1. file_type = map_image
   Case: Referencing a file_id without a reference role.

2. file_type = vr_image
   Case: Referencing a file_id without a reference role.

3. file_type = annotation
   Case: Referencing a file_id without a reference role.

The API behavior should be:

- If permission count is 0:
  {
    "success": false,
    "error_code": "permission_denied",
    "message": "ファイルを取得する権限がありません",
    "errors": null,
    "data": null
  }

- If permission count is 1 or more:
  {
    "success": true,
    "error_code": null,
    "message": "ok",
    "data": {
      "file_id": 1,
      "file_url": "...signed MinIO URL...",
      "file_name": "...",
      "file_type": "...",
      "file_size": 123,
      "created_at": "...",
      "expires_at": "..."
    }
  }


1-3 リクエストパラメータ（バイナリファイルの送信時）
------------------------------------------------------

Purpose:
Test file upload using binary file data.

API:

  POST http://localhost:8081/files

Required header:

  Authorization: Bearer <access_token>

Important:
Do not manually type Content-Type in Postman.
When Body is form-data, Postman automatically creates:

  Content-Type: multipart/form-data; boundary=...

If you manually set Content-Type incorrectly, Spring Boot may return:

  415 Unsupported Media Type

Postman setup:

1. Method:

   POST

2. URL:

   http://localhost:8081/files

3. Headers:

   Authorization: Bearer <access_token>

4. Body:

   Select form-data.

5. Add these rows:

   key        type    value
   file_type  Text    annotation
   file       File    select a real file, for example example_aws_s3_minio.pdf

Allowed file_type values:

  annotation
  vr_image
  map_image

Example for annotation upload:

  file_type = annotation
  file      = example_aws_s3_minio.pdf

Expected success response:

{
  "success": true,
  "error_code": null,
  "message": "ok",
  "data": {
    "file_id": 5012,
    "file_name": "example_aws_s3_minio.pdf",
    "file_path": "/5012",
    "file_type": "annotation",
    "file_size": 2439,
    "s3_bucket": "http://localhost:9000/vr-view-bucket",
    "created_at": "2026/07/22 10:29:35"
  }
}

DB check after upload:

SELECT
  file_id,
  file_name,
  file_path,
  file_type,
  file_size,
  s3_bucket,
  created_at,
  deleted_at
FROM files
WHERE file_id = <response_file_id>;

Expected DB values:

  file_id    = response data.file_id
  file_name  = uploaded original file name
  file_path  = "/" + response file_id
  file_type  = request file_type
  file_size  = uploaded file size in bytes
  s3_bucket  = MinIO bucket URL
  deleted_at = NULL

MinIO check after upload:

1. Open MinIO console:

   http://localhost:9001

2. Open bucket:

   vr-view-bucket

3. Confirm the uploaded file exists under:

   <file_id>/<file_name>

Example:

  5012/example_aws_s3_minio.pdf

Validation error cases:

1. Missing file:

   Body has file_type, but no file.

   Expected:

   {
     "success": false,
     "error_code": "validation_error",
     "message": "リクエストパラメータが不正です",
     "errors": {
       "file": [
         "この項目は必須です"
       ]
     },
     "data": null
   }

2. Missing or invalid file_type:

   file_type is empty, or not one of annotation / vr_image / map_image.

   Expected:

   {
     "success": false,
     "error_code": "validation_error",
     "message": "リクエストパラメータが不正です",
     "errors": {
       "file_type": [
         "annotation、vr_image、map_image のいずれかを指定してください"
       ]
     },
     "data": null
   }

3. Wrong Content-Type:

   Example:
   Body is raw JSON, or Header Content-Type is application/json.

   Expected:
   Spring Boot can return 415 Unsupported Media Type because POST /files only accepts multipart/form-data.


Important Concept
-----------------
The user is non-SYS_ADMIN.

The user has this Keycloak client role:

  viewer

The API reads the user role from the access token. Therefore, after assigning the role in Keycloak, always get a new access token.

For this guide, the database role value must match the token role exactly:

  facility_role_permission.keycloak_role_id = 'viewer'


Required MinIO Setup
--------------------
Start MinIO before testing:

Mac:

  minio server ~/minio-data --address ":9000" --console-address ":9001"

Windows:

  minio.exe server --address :9000 --console-address :9001 data

Application setting must use API port 9000:

  minio.endpoint=http://localhost:9000
  minio.bucket=vr-view-bucket

The console/browser URL is port 9001. Do not use 9001 in Spring Boot config.

Create Required PostgreSQL Tables
---------------------------------
Run this SQL inside database:

  vr_view_db

If tables already exist, this SQL will not recreate them.

------------------------------------------------------------

CREATE TABLE IF NOT EXISTS facilities_tree (
    facility_id BIGSERIAL PRIMARY KEY,
    parent_id BIGINT REFERENCES facilities_tree(facility_id),
    facility_name VARCHAR(128) NOT NULL,
    sort_order INTEGER NOT NULL DEFAULT 10,
    tree_level INTEGER,
    is_leaf BOOLEAN NOT NULL DEFAULT TRUE,
    public_flag CHAR(2) NOT NULL DEFAULT '01',
    outer_flag BOOLEAN NOT NULL DEFAULT FALSE,
    facility_description VARCHAR(1000),
    authority_setting_flg BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS files (
    file_id BIGSERIAL PRIMARY KEY,
    file_name VARCHAR(128) NOT NULL,
    file_path VARCHAR(256),
    file_type VARCHAR(32) NOT NULL,
    file_size BIGINT NOT NULL,
    s3_bucket VARCHAR(512) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS maps (
    map_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    file_id BIGINT NOT NULL REFERENCES files(file_id),
    map_name VARCHAR(256) NOT NULL,
    image_width INTEGER NOT NULL,
    image_height INTEGER NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS facility_images (
    image_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    file_id BIGINT NOT NULL REFERENCES files(file_id),
    image_name VARCHAR(256) NOT NULL,
    shooting_height_m DECIMAL(6,3) NOT NULL,
    ceiling_height_m DECIMAL(6,3) NOT NULL,
    guid VARCHAR(64) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS annotations (
    annotation_id BIGSERIAL PRIMARY KEY,
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    yaw DOUBLE PRECISION,
    pitch DOUBLE PRECISION,
    annotation_type CHAR(2) NOT NULL DEFAULT '01',
    annotation_title VARCHAR(256) NOT NULL,
    annotation_content TEXT,
    display_expire_type VARCHAR(16),
    display_expire_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by VARCHAR(128) NOT NULL,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS annotation_files (
    annotation_file_id BIGSERIAL PRIMARY KEY,
    annotation_id BIGINT NOT NULL REFERENCES annotations(annotation_id),
    file_id BIGINT NOT NULL REFERENCES files(file_id)
);

CREATE TABLE IF NOT EXISTS permission_master (
    permission_id BIGSERIAL PRIMARY KEY,
    permission_name VARCHAR(128) NOT NULL UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS facility_role_permission (
    facility_role_permission_id BIGSERIAL PRIMARY KEY,
    permission_id BIGINT NOT NULL REFERENCES permission_master(permission_id),
    facility_id BIGINT NOT NULL REFERENCES facilities_tree(facility_id),
    keycloak_role_id VARCHAR(64) NOT NULL,
    keycloak_role_name VARCHAR(128) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMPTZ
);

Troubleshooting checklist for permission_denied:

1. The access token must be new.
   After assigning the Keycloak role, get a new token.

2. The token role and DB role must match exactly.
   Example:
   /api/keycloak/me returns:
     "viewer"
   Then DB must use:
     facility_role_permission.keycloak_role_id = 'viewer'

3. The file_type decides which permission check is used.
   map_image  -> permission_id = 20
   vr_image   -> permission_id = 20
   annotation -> permission_id = 30 or 40

4. For annotation files, annotation_files must link the file to an annotation.
   Required chain:
     files.file_id
       -> annotation_files.file_id
       -> annotation_files.annotation_id
       -> annotations.annotation_id
       -> annotations.facility_id
       -> facility_role_permission.facility_id

5. annotation_files does not need deleted_at.
   According to DB設計.md, it is only a link table:
     annotation_file_id
     annotation_id
     file_id

6. If GET /files/{fileId} still returns permission_denied, run Step F.
   If Step F returns 0 rows, the DB permission chain is still incomplete.

Troubleshooting
---------------
1. If SQL count is 1 but API still returns permission_denied:
   - Get a new access token.
   - Check /api/keycloak/me shows "viewer".
   - Restart Spring Boot after code changes.

2. If SQL count is 0:
   - Check files.file_type.
   - Check relation table:
     map_image: maps
     vr_image: facility_images
     annotation: annotation_files and annotations
   - Check facility_role_permission.keycloak_role_id exactly matches token role.

3. If GET returns file_download_failed:
   - Check MinIO is running.
   - Check minio.endpoint uses port 9000.
   - Check bucket exists.

4. If old uploaded files do not open:
   - Re-upload new files with the current code.
   - Current code expects MinIO object path:
       {file_id}/{file_name}
     Example:
       101/test-map.png
