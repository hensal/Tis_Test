facilities一覧取得GETparent指定
================================

1 List GET direct child facilities, SYS_ADMIN
=============================================

Excel section:
一覧取得（直下施設一覧）
GET /facilities?parentId=TopレベルファシリティのID
（SYS_ADMINロールを持つユーザ）

Endpoint:
GET /facilities?parentId=<top_facility_id>

User:
SYS_ADMIN access token

Note:
The Excel uses these names:
- PARENT_FACILITY_ID = parent_id
- NODE_NAME = facility_name
- PUBLIC_FLAG = public_flag / response publish_mode
- OUTER_FLAG = outer_flag / response is_outdoor

Find a Top facility ID first
----------------------------

Use a facility that has child facilities.

SQL:

SELECT
  p.facility_id AS top_facility_id,
  p.facility_name AS top_facility_name,
  COUNT(c.facility_id) AS child_count
FROM facilities_tree p
JOIN facilities_tree c
  ON c.parent_id = p.facility_id
WHERE p.deleted_at IS NULL
  AND c.deleted_at IS NULL
GROUP BY p.facility_id, p.facility_name
ORDER BY p.facility_id;

Pick one `top_facility_id` where `child_count >= 1`.

Postman request
---------------

Method:
GET

URL:
http://localhost:8081/facilities?parentId=<top_facility_id>

Headers:
Authorization: Bearer <SYS_ADMIN access_token>

Expected base response:

{
  "success": true,
  "error_code": null,
  "message": "ok",
  "data": [
    {
      "facility_id": 1001,
      "parent_id": 1,

      "publish_mode": "02",
      "is_outdoor": false,

8 List GET direct child facilities of lowest-level facility, USER01
==================================================================

Japanese test item:
一覧取得（直下施設一覧）
GET /facilities?parentId=最下層のファシリティのID
（USER01）

Goal:
Use a USER01 token and request the direct children of a lowest-level facility.
Because the selected facility is already the lowest level, it has no child facilities.
The API should return a normal success response with zero data rows.

Important:
For USER01 tests, first confirm which Keycloak roles the login user has.

Postman:
GET http://localhost:8081/api/keycloak/me
Authorization: Bearer <USER01 access_token>

Example expected role:
R_USER01

Step 1: find a lowest-level facility visible to USER01
------------------------------------------------------
Run this SQL first.

SQL:

SELECT DISTINCT
  ft.facility_id AS lowest_level_facility_id,
  ft.parent_id,
  ft.facility_name,
  ft.tree_level
FROM facilities_tree ft
JOIN facility_role_permission frp
  ON frp.facility_id = ft.facility_id
WHERE ft.deleted_at IS NULL
  AND ft.is_leaf = TRUE
  AND frp.deleted_at IS NULL
  AND frp.keycloak_role_id IN ('R_USER01')
ORDER BY ft.tree_level DESC, ft.facility_id ASC;

Use one returned lowest_level_facility_id as <lowest_level_facility_id>.

Step 2: confirm this facility has no child facilities
----------------------------------------------------
SQL:

SELECT COUNT(*) AS child_count
FROM facilities_tree
WHERE parent_id = <lowest_level_facility_id>

If child_count is not 0, choose another lowest_level_facility_id from Step 1.

Headers:

Expected:

Find a top facility whose children are visible to USER01
-------------------------------------------------------

SQL:

SELECT
  p.facility_id AS top_facility_id,
  p.facility_name AS top_facility_name,
  COUNT(DISTINCT c.facility_id) AS visible_child_count
FROM facilities_tree p
JOIN facilities_tree c
  ON c.parent_id = p.facility_id
JOIN facility_role_permission frp
  ON frp.facility_id = c.facility_id
WHERE p.deleted_at IS NULL
  AND c.deleted_at IS NULL
  AND frp.deleted_at IS NULL
  AND frp.keycloak_role_id IN ('R_USER01')
GROUP BY p.facility_id, p.facility_name
HAVING COUNT(DISTINCT c.facility_id) > 0
ORDER BY p.facility_id;
